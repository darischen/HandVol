# HandVol Windows Installer

This directory holds the source files for building `HandVol-Installer.exe` — a
self-contained installer that bundles an embeddable Python runtime, the
MediaPipe model, and the HandVol code, then registers a Windows Task Scheduler
entry so HandVol runs automatically.

## Prerequisites

- Windows 10 (build 19041+) or Windows 11, x64
- Python 3.11+ (to run the build script)
- NSIS 3.x — install from https://nsis.sourceforge.io/Download and make sure
  `makensis.exe` is on your PATH
- Git with Git LFS (`git lfs install`), so `models/gesture_recognizer.task`
  resolves to the real ~8 MB model rather than an LFS pointer

## Building

From the **repo root**:

```powershell
git lfs pull            # ensure the model is the real file, not a pointer
python build_installer.py
```

The script (`../build_installer.py`) will:

1. Download embeddable Python 3.11 and `get-pip.py` (cached in `cache/`)
2. Extract Python to `build/python/`
3. **Patch `pythonXY._pth`** to enable `site` and add `Lib\site-packages`
   (the embeddable distribution disables both by default)
4. **Bootstrap pip** into the embeddable runtime via `get-pip.py`
5. `pip install -r requirements.txt` into the bundled runtime
6. Copy the MediaPipe model and HandVol source into `build/`
7. Compile the NSIS script to `dist/HandVol-Installer.exe`

Output: `dist/HandVol-Installer.exe` (roughly 60–90 MB — dominated by the
mediapipe/opencv/numpy wheels, not the model).

First run takes a few minutes (download + pip install). Subsequent runs reuse
`cache/` and are faster.

## Files

- `handvol-installer.nsi` — NSIS installer script (wizard UI, file extraction,
  Add/Remove Programs registration, uninstaller). Compiled with `makensis`
  invoked from the **repo root**, so its relative paths (`build\...`,
  `dist\...`) are repo-root-relative.
- `install_task.bat` — creates the `HandVol` Task Scheduler entry; called by the
  installer with `INSTDIR` as its argument.
- `uninstall_task.bat` — removes the `HandVol` task; called by the uninstaller.

## Installed layout (`C:\Program Files\HandVol\`)

```
python\           embeddable Python 3.11 + Lib\site-packages (deps)
models\gesture_recognizer.task
handvol\          HandVol package
handvol.pyw       entry point
requirements.txt
install_task.bat / uninstall_task.bat
uninstall.exe     generated by NSIS
```

`handvol/capture.py` resolves the model as `<parent.parent>/models/...`, which
maps to `C:\Program Files\HandVol\models\gesture_recognizer.task` in this layout.

## Scheduled task trigger

`install_task.bat` registers the task with `/sc onlogon /rl highest`. This
deviates from the design spec's `onstart` on purpose: `onstart` runs in the
non-interactive Session 0 before any user logs in, where a system-tray GUI that
needs the user's desktop, webcam, and audio session cannot function. `onlogon`
runs HandVol in the user session at login, which is what a tray app needs.
`/rl highest` keeps it elevated for audio control.

## Testing

### Fresh Windows VM

1. Copy `dist\HandVol-Installer.exe` to the VM
2. Run it (admin elevation is requested automatically)
3. Verify: wizard appears → files land in `C:\Program Files\HandVol\` → a
   `HandVol` task exists in Task Scheduler → optional run-now / desktop shortcut
   work → MediaPipe model loads on first run

### Uninstall

1. Settings → Apps → Installed apps → HandVol → Uninstall (or run
   `C:\Program Files\HandVol\uninstall.exe`)
2. Verify the install dir, the `HandVol` task, the desktop shortcut, and the
   `...\Uninstall\HandVol` registry key are all gone

## Troubleshooting

- **"makensis not found on PATH"** — install NSIS 3.x and add it to PATH.
- **"looks like an unresolved Git LFS pointer"** — run `git lfs pull`.
- **"get-pip failed" / "pip install failed"** — check your internet connection;
  delete `cache\` and retry. mediapipe/opencv wheels are large.
- **App doesn't appear after boot** — see the scheduled-task note above
  (`onstart` vs `onlogon`).
